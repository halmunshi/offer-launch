## Design Context

### Users
- Solo coaches/consultants, realtors, ecommerce brands, marketers, agencies
- Primary workflow context is desktop work sessions
- Core job: shape and refine offer details quickly, then move into funnel creation and optimization

### Brand Personality
- Forward
- Innovative
- Clear
- Emotional goals: confidence and momentum

### Aesthetic Direction
- Keep current OfferLaunch visual language
- Light-first product experience
- Avoid template-like generic SaaS patterns

### Design Principles
- Prioritize clarity first: each screen should make the next action obvious
- Maintain momentum: reduce friction in high-frequency edit and save flows
- Keep interactions intentional: hover/focus/loading states should be consistent and reassuring
- Preserve visual continuity: build from existing spacing, type, and color patterns
- Ship polished utility over decoration: every visual change must improve comprehension or confidence

### Accessibility Baseline
- WCAG AA contrast and keyboard support as default bar
- Respect reduced-motion preferences for non-essential motion
